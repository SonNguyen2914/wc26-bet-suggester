# V9.1.2 execution-fidelity hotfix — fourth-evaluation P0s

**July 24, 2026 — a DISCLOSED protocol deviation from the pre-slate
freeze.** A fourth independent evaluation of `wc26-project-2026-07-24-
V9.1.zip` found a **critical order-book correctness defect** and several
execution-fidelity + governance gaps. Because the depth defect corrupts
*stored* evidence (the frozen T-10 books), and the first slate had zero
locks (the last clean moment), the freeze was deliberately re-opened for
this correctness hotfix. New baseline tag: **`mls-shadow-v1.2`**. Money
stays locked; nothing here changes the model, thresholds, lock timing, or
market-family policy — only execution fidelity, audit strength, and
governance.

## Critical / high — fixed

| # | Finding | Fix |
|---|---------|-----|
| **F1** | Depth parser kept the WRONG end of Kalshi's ascending bid arrays (`[:10]` = the worst 10, dropping the best bids) | `_depth_levels` now sorts each side by price and keeps the BEST N — correct regardless of provider array order. Regression test uses 12 ascending levels and asserts the highest bid (best implied ask) survives. Policy: `best_10_each_side`. |
| **F2** | Paper execution used rounded integer cents/contracts, discarding the exact fixed-point values the archive retains | `yes_buy_ladder`/`simulate_fill` and the whole fill path are **Decimal end-to-end**, walking the exact `price_dollars`/`size_fp`. Exact economics stored on `paper_signal`/`paper_fill` (`*_dollars`, `filled_contracts_fp`); cents are display only. |
| **F3** | Order-level fee used binary float → `ceil` overcharged 1¢ at some prices (100@$0.10 gave 64¢ vs 63.00) | `order_fee_dollars` is exact centicent `ceil` in Decimal. Versioned `FEE_POLICY` with taker/maker/series fields; **general taker only** is populated and labelled — series overrides, maker, and exit fees remain declared-not-modeled. |
| **F4** | Audit checked that an approval hash / engine signature were PRESENT, not valid — a bogus hash + bogus engine passed `all_pass` | The audit now **recomputes** the approval-decision hash from its stored canonical `decision_document` and **compares** the lock's frozen engine signature against the current engine. `approval_decision_hash_valid` and `engine_signature_matches_current` are required for `all_pass`. Regression test feeds the evaluator's bogus values and asserts failure. |
| **F5** | Engine signature fingerprinted only selected constants + numpy; a code change could pass the replay guard | `engine_signature` now hashes the **source of the model/simulator modules** (`model_mls`, `simulator`, `xg_model`, `features`) plus code revision, python, and numpy. Artifact bumped to `model-input-v3`. (A container/wheel digest is the stronger form; a source-tree digest is the in-process version.) |

## Medium — fixed

| # | Finding | Fix |
|---|---------|-----|
| **F6** | Timestamp-less books qualified execution-ready via the capture-time fallback | `execution_ready` now requires **provider-timestamp** freshness. Capture-time is honest *research* freshness (recorded) but **not executable**. Capture-completeness is unaffected. |
| **F7** | Paper readiness ignored active data-driven kill switches (e.g. DAILY_LOSS_LIMIT) | `/api/ready` now splits `paper_engine_operational` (can settle/reconcile) from `paper_new_entries_allowed` (requires zero active kill switches); exposes `paper_kill_switches`. |
| **F8** | Approval could be recomputed on mutable DB at each boot → drift | Boot **loads** the active approval decision (does not recompute); a re-evaluation is an explicit `force=True` operator action. A pre-v1.2 row missing its canonical document is healed once (its hash still verifies). |
| **F9** | A canonical run could be created without an approval decision, caught only after by the audit | `t10_locks` refuses to attempt a lock without an approved decision, and `_write_run` structurally refuses to create a canonical run when none is referenced. |
| **F10** | Registry-discovery completeness was transient | Each sweep persists a durable `registry_discovery` record (complete / truncated-series / counts). |
| **F12** | Corpus publication had a runtime overwrite escape | Removed — re-publishing a version is always refused; a mistaken publish is corrected by a deliberate migration. |

## Claims — now accurate (or narrowed to what the code proves)

- **"Audit enforces a matching engine"** — now TRUE (F4): `all_pass`
  requires the engine signature to match the current engine and the
  approval hash to recompute.
- **"Engine-matched replay"** — now a **source-tree + runtime** signature
  (F5), not just constants. Still not a container/wheel digest — that is
  the declared stronger form.
- **"Exact market execution"** — TRUE for paper economics now (F2), over
  **best-10-per-side** bounded depth (F1).
- **Paper layer** — "bounded-depth (best-10) taker-entry, hold-to-
  settlement paper simulation with **exact general-taker fees**." Still
  not full execution-quality: maker/taker & series/exit fees, bid-side
  exits, and order-lifecycle remain unmodeled.

## Explicitly NOT closed here (deferred, need external infra / data / gates)

- Per-series & maker/taker fee **data** (framework is in place; values not
  populated).
- Container/wheel digest for the engine signature (source-tree digest used
  instead).
- Full raw provider payloads in **object storage** (F11) — still truncated
  in PG.
- Corpus bytes in object storage / route cost controls (F12 residue).
- Model-methodology: nested/holdout selection and evaluating the exact
  deployed stochastic distribution (would change the approval computation —
  not done mid-freeze).
- Everything real-money (prospective evidence, availability plane, capped
  pilot) — unchanged: **not approved**.
